# 版本路线

版本范围按“先建立可用闭环，再快速扩展录入效率”的原则安排。

## v0.1：手动录入 MVP

- 状态：代码实现完成，等待真实 Supabase 项目联调与手动验收
- 手动添加商品
- 手动添加库存批次
- 支持生产日期 + 保质期计算到期日
- 支持直接填写保质期至
- 首页按库存批次的到期日排序
- 显示已过期、7 天内到期、30 天内到期、正常等状态
- 支持某个库存批次快捷减少 1
- 支持手动修改库存批次数量
- 数量为 0 时可标记为已消耗
- 使用 Supabase Anonymous Sign-in 打开即用
- 使用 `user_id` 和 RLS 隔离匿名用户数据

进入 v0.2 前应完成：

- 在真实 Supabase 项目执行 schema。
- 开启 Anonymous Sign-ins。
- 完成 `docs/TESTING.md` 中的 7 项手动验收。
- 补充匿名账号绑定邮箱/恢复策略的产品设计，但不阻塞扫码原型。

## v0.2：扫码和商品信息自动填充

- 支持摄像头扫描条形码
- 支持手动输入条形码
- 通过开放商品数据库查询商品名和图片
- 查不到时允许手动补充商品名和图片
- 查询到的商品信息保存到 `products`，避免重复录入
- 设计匿名账号绑定邮箱入口，为跨设备同步和账号恢复做准备

## v0.3：批次和筛选体验优化

- 商品详情页展示该商品下所有库存批次
- 支持按分类、储存位置、临期状态筛选
- 支持搜索商品名
- 支持编辑批次信息

## v0.4：提醒能力

- 支持 PWA
- 支持临期提醒设置
- 可考虑 Bark / ntfy / 浏览器通知

## v0.5：备份和导入导出

- CSV 导出
- JSON 备份
- 批量导入
