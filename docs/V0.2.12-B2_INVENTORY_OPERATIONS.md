# v0.2.12-B2 库存操作规格

## 目标

在商品详情的 `inventory-operation` 状态中完成家庭食品场景的真实库存操作：

- 展示当前批次库存数量和保质期。
- 从当前商品新增库存，不重新扫码、不重复创建 `products`。
- 同 `product_id`、同 `expiry_date`、`active` 批次直接累加数量；不同日期创建新的 `inventory_batches`。
- 消耗库存先进入确认状态，确认后才更新数量；取消不写入。
- 数量降为 0 时保持原有 `status`，只有显式确认“标记为已消耗”才更新为 `consumed`。

## 技术边界

- 复用现有 React、Vite、Vitest 和 Supabase 客户端，不新增依赖。
- 只写入 `inventory_batches.quantity`、`inventory_batches.status`，以及新增批次所需的既有批次字段。
- 不修改 `products` 数据结构、Supabase schema、migration、Auth、RLS、Storage 或环境变量。
- 新增批次沿用当前批次的 `unit` 和 `storage_location`，用户只输入数量与保质期至日期。

## 验收标准

- `inventory-operation` 显示当前数量、当前批次保质期，并提供新增库存、消耗库存入口。
- 新增库存能验证同日期合并与不同日期新建批次，且商品信息只读带入。
- 消耗确认默认数量为 1；取消不写入，确认只更新选中批次，不能负库存或超过当前库存，重复提交被禁用。
- 数量为 0 不自动变为 `consumed`；显式确认后才更新 status。
- `npm test`、`npm run build`、`git diff --check` 全部通过。
