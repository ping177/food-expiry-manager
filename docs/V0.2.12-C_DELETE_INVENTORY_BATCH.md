# v0.2.12-C Delete Inventory Batch 规格

## 目标

让用户在商品详情的库存操作中，删除错误录入或测试用的当前
`inventory_batches` 记录。

## 技术与命令

- React、Vite、Vitest、Supabase JavaScript client；不新增依赖。
- 测试：`npm test`
- 构建：`npm run build`
- Diff 检查：`git diff --check`

## 实现范围

- 删除入口仅存在于 `InventoryOperationPanel`。
- 点击后必须二次确认；取消不执行数据库操作，确认期间禁止重复提交。
- `App` 仅调用 `inventory_batches` 的 delete，并按 `id` 与当前 `user_id` 限定。
- 成功后清除选中 batch、返回库存列表、刷新 active batches；0 行删除或 Supabase
  错误都显示失败信息，不能报成功。

## 数据边界

- 删除对象仅为当前 `inventory_batches` 记录。
- 保留 `products`、`user_image_url`、Storage 图片和其他 batch。
- 不修改 schema、migration、Auth、RLS、Storage、环境变量或依赖。

## 测试策略

- `src/lib/inventory.test.js` 覆盖删除确认生成当前 batch id 与取消无写入 payload。
- `src/components/BatchDetail.test.jsx` 覆盖入口仅在 inventory-operation。
- `src/components/InventoryOperationPanel.test.jsx` 覆盖确认 UI / 回调接线契约。
- `src/App.test.jsx` 覆盖 delete 目标、owner 条件、0 行失败处理与列表返回接线。
- 最终运行完整测试、生产构建和 diff 检查；真实 Supabase 删除 / RLS 验收另行执行。

2. 在库存操作面板实现危险区与二次确认，并在 App 实现删除及刷新。
3. 更新版本记录并运行测试、构建与 diff 检查。
